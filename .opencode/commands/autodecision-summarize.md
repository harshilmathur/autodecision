# /autodecision-summarize

Compress a Decision Brief into a one-page summary for sharing with teams, boards, or stakeholders. Run on: $ARGUMENTS

## Usage

```
/autodecision-summarize {slug}
/autodecision-summarize pricing-cut-20pct-full
```

## Bootstrap

Read via `read`:
1. `.claude/skills/autodecision/SKILL.md`
2. `.opencode/host-adapter.md`

## Execute

1. Parse `{slug}` from `$ARGUMENTS`. Read `~/.autodecision/runs/{slug}/DECISION-BRIEF.md`.

2. If the file doesn't exist, use `glob` on `~/.autodecision/runs/*/DECISION-BRIEF.md` to list available runs, then use `question` to let the user pick one.

3. Extract and compress into this exact format:

```markdown
# One-Page Summary: {decision statement}

**Recommendation:** {action — one sentence, no hedging}
**Confidence:** {level} | **Tilt:** {tilt} | **Converged:** {yes/no}

---

## Why This Is the Right Move
- {strongest argument 1 — grounded in data, specific numbers with tags}
- {strongest argument 2}
- {strongest argument 3}

## Top Risks
- {risk 1 — probability, what triggers it}
- {risk 2}
- {risk 3}

## What Must Be True (load-bearing assumptions)

| Assumption | State | If Wrong |
|-----------|-------|----------|
| {assumption 1} | SOLID/SHAKEABLE/FRAGILE | {what happens} |
| {assumption 2} | | |
| {assumption 3} | | |

## Action Plan

| Phase | Timeline | Key Action | Kill Criteria |
|-------|----------|-----------|---------------|
| 1 | {when} | {what} | {stop if} |
| 2 | {when} | {what} | {stop if} |
| 3 | {when} | {what} | {stop if} |

## Watch For
- {monitoring signal 1 — frequency, trigger}
- {monitoring signal 2}
- {monitoring signal 3}

---

*Full analysis ({N} pages): ~/.autodecision/runs/{slug}/DECISION-BRIEF.md*
*Generated by Auto-Decision Engine | {date}*
```

4. Write to `~/.autodecision/runs/{slug}/SUMMARY.md`.

5. Print to the conversation.

6. Offer next steps via `question`:

```
Share this summary?
  A) Publish — run /autodecision-publish (PDF → Notion, email, gist, Slack, Drive, or local)
  B) Copy summary to current directory
  C) Copy full brief to current directory
  D) Skip
```

- **A:** invoke the publish flow for the current slug (tell the user to run `/autodecision-publish {slug} --summary`).
- **B:** via `bash`: `cp ~/.autodecision/runs/{slug}/SUMMARY.md ./{slug}-SUMMARY.md`
- **C:** via `bash`: `cp ~/.autodecision/runs/{slug}/DECISION-BRIEF.md ./{slug}-DECISION-BRIEF.md`

## Rules

- All human-readable output rules apply. **No `snake_case` in prose.** No backticked `effect_id`s.
- The summary must stand alone — a reader understands the recommendation without ever reading the full brief.
- Max ~40 lines of content excluding table formatting.
- Every bullet is specific and grounded. No "various factors" or "multiple considerations."
- Preserve the source tags `[G#]`, `[D#]`, `[U#]`, `[C#:persona]` wherever numbers appear. Summaries still need provenance.
