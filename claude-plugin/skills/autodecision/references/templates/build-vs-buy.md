# Template: Build vs Buy

Use when deciding whether to build a capability in-house or buy/integrate
an external solution — SaaS tools, infrastructure, acquired technology.

## Pre-populated Sub-Questions

1. **Core vs context:** Is this capability core to our competitive advantage, or operational context?
2. **Total cost of ownership:** What's the real cost over 3 years — build (eng time + maintenance) vs buy (license + integration + vendor risk)?
3. **Time to value:** How fast do we need this? Build takes months; buy takes weeks.
4. **Control and customization:** Do we need deep customization, or is 80% of off-the-shelf good enough?
5. **Switching costs:** If we buy now, how hard is it to switch later? If we build, how hard to maintain?

## Pre-populated Constraints

- Must be operational within [user specifies] timeline
- Engineering team capacity: [user specifies] available engineers
- Budget: [user specifies] annual budget for buy option

## Domain-Specific Personas Enhancement

- **Growth Optimist:** Focus on strategic advantage of owning the capability, future optionality
- **Risk Pessimist:** Focus on maintenance burden, key-person risk, opportunity cost of eng time
- **Competitor Strategist:** Focus on what competitors do, whether this is a differentiator
- **Regulator:** Focus on data handling, vendor compliance certifications, SLA guarantees
- **Customer Advocate:** Focus on user experience quality, integration seamlessness, reliability

## Key Assumptions to Watch

- Engineering time estimates (historically underestimated 2-3x)
- Vendor stability and longevity (evaluate funding, revenue, trajectory)
- Maintenance cost of built solution (the hidden 70% of total cost)
- Integration complexity for buy option
